Nexus Calendar

Unified, collaborative calendar system with ICS imports + Google Calendar sync

← back

Overview

Nexus Calendar addresses the fragmentation of modern scheduling across Google Calendar, iCloud, and Outlook. Many users struggle with overlapping events, manual updates, and poor collaboration between platforms. Nexus Calendar provides a centralized interface that mirrors the familiarity of a physical calendar while supporting shared scheduling and imports from multiple providers.

We built this as a web-based system for cross-platform access and rapid deployment, with a foundation that can support deeper integrations in the future.

Architecture

The system pulls calendar events from a Firebase database and displays them on a touchscreen connected to a Raspberry Pi. Updates to events flow through the same database, including group calendars and shared changes.

Key Features

  • Unified calendar interface designed to mimic a physical calendar experience
  • Group calendars for shared scheduling and collaboration
  • Google Calendar synchronization via API (real-time integration)
  • ICS file import for Outlook / iCloud calendars (manual upload workflow)
  • Fast updates: changes reflected across devices within ~30 seconds (avg ~10s)

Tech Stack

  • Frontend: Web UI (responsive), calendar views + event creation flow
  • Backend / Auth: Firebase Authentication
  • Database: Firestore (events stored under user calendar collections)
  • Integrations: Google Calendar API + OAuth2; ICS parsing for other providers
  • Hardware: Raspberry Pi + touchscreen display

Data Model

  • Users: stored with email, name, Nexus ID, Firebase UID, timestamp
  • Calendars: provider-specific (Google / iCloud / Outlook) collections
  • Events: title, start time, end time, location, description
  • Groups: group metadata, members, invites, shared calendar behavior

Core Flows

Authentication + User Setup

  • Register user → create Firebase Auth user → generate Nexus user ID → store user record in Firestore
  • Login → validate credentials → look up user → return token + Nexus user ID

Calendar Ingestion

  • Google: OAuth2 → pull events via Google Calendar API → batch store to Firestore
  • Outlook/iCloud: user uploads ICS file → parse events → store to provider path in Firestore

Groups

  • Create group, invite members, accept/decline invites
  • Group members view/edit/upload events with shared visibility

My Role

I focused on front-end development: building the calendar interface and user experience. I worked on UI structure, interactions (creating/editing events), and ensuring the interface stayed clean and intuitive across devices.

Challenges & Tradeoffs

  • API restrictions: Microsoft and Apple APIs were inaccessible, so we pivoted to manual ICS imports for Outlook and iCloud while keeping real-time Google sync.
  • Platform decision: we shifted from an app concept to a website for cross-platform access and easier deployment within time/budget.
  • Offline requirement: became unattainable after choosing a website (web use requires internet).
  • Notes/highlights: missed due to project management oversight during integration work.

Testing Results

  • Calendars connect and events appear in the UI (Pass)
  • No duplicates / no lost events from source calendars (Pass)
  • Updates appear across devices within 30 seconds (Pass; average ~10 seconds)
  • Notes/highlights persistence (Fail)
