Indoor Cat Tracker (ICT)

BLE-based indoor positioning system with real-time visualization

Overview

Indoor Cat Tracker is a scalable IoT system designed to help pet owners monitor their cat’s location inside their home. Using BLE beacons, a wearable receiver, and a cloud-hosted web interface, the system estimates position in real time and visualizes movement patterns across sessions.

The goal was to combine hardware sensing, data ingestion, and web-based analytics into a single usable system.

Architecture

System Architecture (Components)

  • BLE Beacons + Receiver (ESP-32): Beacons broadcast signals; receiver estimates distance using RSSI.
  • Raspberry Pi Bridge: Aggregates BLE readings and forwards batched data over Wi-Fi.
  • Cloud Backend: Stores session and position data; serves the web UI.
  • Web Dashboard: Displays live location data, session heatmaps, and playback controls.

Data & Analytics

  • Location points recorded every 1–2 seconds
  • Session-based tracking (start / stop / review past sessions)
  • Heatmap visualization of frequently visited locations
  • Historical sessions stored per user for behavior analysis
  • Boundary-based alerts notify users when the cat leaves the beacon range

Tech Stack

  • Embedded / Hardware: ESP-32 (BLE 5.3), Arduino C
  • Bridge: Raspberry Pi, Node.js (BLE ingestion + HTTP forwarding)
  • Backend: Python, Flask, SQLite
  • Frontend: HTML, JavaScript (dynamic heatmaps & session playback)
  • Deployment: AWS EC2
  • Protocols: BLE + Wi-Fi

My Role

  • Built the initial web server and key API endpoints to support data flow and session handling
  • Developed the frontend interface for live tracking and session playback
  • Focused on usability: interactive UI that works in any browser without client installs
  • Helped translate session data into interpretable heatmaps and timeline playback

Challenges & Tradeoffs

  • BLE RSSI introduces noise and limits positional accuracy
  • Used trilateration after angle-of-arrival features weren’t available in ESP-32 core libraries
  • Improved results by tuning transmission power and increasing beacon count
  • Chose SQLite for simplicity given low write frequency and lightweight storage needs

Security & Privacy

  • User authentication via flask_login
  • Minimized data collection (timestamps and position only)
  • Server-side storage and processing to reduce exposure

Outcome

Indoor Cat Tracker demonstrated a full end-to-end IoT pipeline: sensors → ingestion → cloud storage → analytics → visualization. The system was technically feasible and extensible, with clear next steps around accuracy and UX polish.
